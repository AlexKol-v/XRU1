# Документация проекта XRU1 — «Тихий узел» (тактика в духе XCOM 2)

Эта папка — **единственный источник истины** по дизайну, плану и правилам проекта.
Код и контент делаются «по документам»; если решение меняется — сначала правится
документ, потом реализация. История переноса из донора — отдельно, в
[../MIGRATION_NOTES.md](../MIGRATION_NOTES.md).

## Как этим пользоваться (мне и агентам)

1. **Начало любой сессии:** открыть [04_ROADMAP.md](04_ROADMAP.md), найти первый
   незакрытый этап, взять из него задачи. Контекст по механикам — в GDD,
   по редакторским шагам — в EDITOR_GUIDE.
2. **Постановка задачи агенту:** «Работаем по docs/04_ROADMAP.md, этап N, задачи
   такие-то. Правила — docs/06_CONVENTIONS.md». Этого достаточно: документы
   самодостаточны.
3. **Любое отклонение от GDD** (упростили механику, поменяли число) — сразу
   фиксируется в GDD, чтобы документ не разошёлся с игрой.
4. **После выполнения** — проставить чекбоксы в ROADMAP и, если менялся C++,
   обновить беклог в 03_CODE_OVERVIEW.
