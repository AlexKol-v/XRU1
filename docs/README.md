# Документация проекта XRU1 — «Тихий узел» (тактика в духе XCOM 2)

Эта папка — **единственный источник истины** по дизайну, плану и правилам проекта.
Код и контент делаются «по документам»; если решение меняется — сначала правится
документ, потом реализация. История переноса из донора — отдельно, в
[../MIGRATION_NOTES.md](../MIGRATION_NOTES.md). Закрытые рабочие чеклисты и логи
прошлых сессий — в [archive/](archive/), они не нужны для текущей работы.

## Как этим пользоваться (мне и агентам)

1. **Начало любой сессии:** открыть [08_HANDOFF.md](08_HANDOFF.md) — там
   актуальный срез «где мы и что дальше». Для плана по этапам — [04_ROADMAP.md](04_ROADMAP.md);
   для текущей фазы отработки геймплея — [09_GAMEPLAY_STATUS.md](09_GAMEPLAY_STATUS.md).
   Контекст по механикам — в GDD, по редакторским шагам — в EDITOR_GUIDE.
2. **Постановка задачи агенту:** «Работаем по docs/04_ROADMAP.md, этап N, задачи
   такие-то» (или «по 09_GAMEPLAY_STATUS.md, блок B.N»). Правила —
   docs/06_CONVENTIONS.md. Этого достаточно: документы самодостаточны.
3. **Любое отклонение от GDD** (упростили механику, поменяли число) — сразу
   фиксируется в GDD, чтобы документ не разошёлся с игрой.
4. **После выполнения** — проставить чекбоксы в ROADMAP (и/или блок в
   GAMEPLAY_STATUS), и, если менялся C++, обновить карту классов/беклог в
   03_CODE_OVERVIEW. Обновить 08_HANDOFF в конце сессии.
