# GDD — «Тихий узел» (XRU1)

Короткая одиночная демо-игра в жанре **пошаговой тактики в духе XCOM 2**.
Референс — XCOM 2 (Firaxis): укрытия, очки действий, overwatch, отряд из
4 классов, цель миссии + эвакуация. Демо = главное меню → хаб с 3D-картой →
обучение (по классам) → боевая миссия «разминирование» → финальный экран.

> Соответствие учебному заданию — раздел 13. Лор и тексты — [02_LORE_SCRIPT.md](02_LORE_SCRIPT.md).

## 2. Столпы дизайна

1. **Узнаваемый XCOM-цикл**: два очка действия, укрытия решают, overwatch наказывает.
2. **Обучение через игру**: туториал по этапам знакомит с КАЖДЫМ классом и его
   способностью на скриптованных ситуациях (см. §5.8 и сценарий).
3. **Миссия — это задача, а не тир**: цель-объект (бомба), таймер ходов,
   эвакуация отряда — как в XCOM.
4. **Ничего лишнего**: нет инвентаря, прокачки, кастомизации, разрушаемости, грид-сетки.

## 3. Игровой цикл

```
Главное меню
 ├─ Новая игра → Выбор сложности → ИНТРО-ВИДЕО (~45 сек, скип кликом) → Хаб
 ├─ Продолжить (если есть сейв) → Хаб (интро не повторяется)
 ├─ Настройки / Об авторе / Выход
Хаб (голографическая 3D-карта, вращение камерой)
 ├─ POI «Полигон “Купол”» (обучение) → L_Tutorial
 └─ POI «Станция “Узел-7”» (миссия) → L_Mission01
     └─ ЗАБЛОКИРОВАНА, пока не пройдено обучение (RequiredCompletedMission =
        "Tutorial"): маркер серый, клик → «Сначала пройдите обучение»
L_Tutorial: поэтапное обучение (медик → танк+снайпер → штурмовик → overwatch)
            → зона эвакуации → результат → хаб
L_Mission01: брифинг → бой → обезвредить бомбу до истечения таймера ходов
             → отряд в зону эвакуации → победа
 ├─ Победа → экран результата → финальный экран демо → Главное меню
 └─ Поражение → экран результата → «Повторить» / «В хаб»
```

Порог 10 минут закрывается с запасом: полное обучение + миссия + меню/хаб.
Туториал можно перепройти из хаба (реиграбельность и добор времени при показе).

## 5. Боевая система — правила

### 5.1 Ходы
- Бой = чередование фаз: **ход игрока → ход врага** (`UTurnManagerSubsystem`).
- В свою фазу игрок активирует юнитов в любом порядке, любое чередование.
- Кнопка **«Завершить ход»** передаёт фазу врагу; фаза врага исполняется
  автоматически (юниты действуют по одному), затем ход возвращается.
- **Условия конца боя** зависят от типа миссии (§5.7): туториал — выполнить все
  шаги и эвакуироваться; миссия — обезвредить бомбу и эвакуировать отряд.
  Поражение всегда: весь отряд мёртв/тяжело ранен, либо (миссия) истёк таймер ходов.

### 5.2 Очки действия (AP)
- У каждого юнита **2 AP** на ход, сбрасываются в начале фазы его стороны.
- Стоимости: перемещение — 1 AP; атака — 1 AP и **завершает активацию юнита**
  (остаток AP сгорает, как в XCOM); Overwatch — 1 AP и завершает активацию;
  Глухая оборона — 1 AP и завершает активацию; Пропустить ход — сжигает все AP;
  взаимодействие с целью миссии (обезвреживание) — 1 AP.
- Способности классов — стоимость в таблице раздела 7.

### 5.3 Перемещение и зона хода
- **Свободное перемещение без грид-сетки** (сознательное упрощение XCOM), но
  бюджет дистанции считается **по длине пути по навмешу**, а не по прямой:
  за 1 AP юнит проходит путь длиной до **MoveRange = 800 см**; обходить стену —
  дороже, чем идти напрямик.
- **Визуализация зоны хода**: у выбранного юнита подсвечивается **область
  достижимости** — обведённая фигура по навмешу (не круг): синяя граница —
  достижимо за 1 AP (путь ≤ 800), жёлтая — за 2 AP (путь ≤ 1600). Реализация:
  выборка полигонов навмеша по path-distance + контур (см. беклог C++ P0).
- Приказ в точку за пределами оплачиваемой зоны не исполняется (клик игнор +
  звук отказа).
- После перемещения юнит автоматически пересчитывает укрытие.

### 5.4 Стрельба и шанс попадания
```
HitChance = BaseAim − CoverDefense(цели против стрелка) + Модификаторы
Clamp: [5% .. 95%]   (кроме скриптовых выстрелов туториала: форс 100% / 0%)
```
| Параметр | Значение |
|---|---|
| BaseAim юнитов игрока | 75 (снайпер 85) |
| BaseAim врагов | 65 |
| CoverDefense half / full | −20 / −40 |
| Фланкирование (укрытие не между стрелком и целью) | укрытие не действует (направленный трейс это даёт автоматически) |
| Глухая оборона | удваивает бонус укрытия цели до её след. хода |
| Штраф реакционного выстрела (overwatch) | −10 к BaseAim |
| Штраф Squadsight (стрельба без собственной LOS) | −10 |
| Дальность стрельбы | 1100 см (снайпер 2000 см), нужна линия видимости |
| Крит/разброс урона | Нет критов; урон фиксированный ± случайные 10% |

Урон: винтовка отряда **25 HP**, снайперская **40 HP**, враги **20 HP**.
Убийство = HP ≤ 0: враги умирают; юниты игрока переходят в состояние
**«тяжело ранен»** (Downed) — лежат, не действуют, поднимаются медиком.
Ранены/мертвы все — поражение.

### 5.5 Укрытия
- Детекция трейсами (`UCoverDetectionComponent`): низкая преграда (высота
  ~60 см) = **half cover**, высокая (~150 см) = **full cover**.
- Укрытие работает **направленно**: бонус только если преграда между юнитом
  и стрелком. UI показывает щит-иконку (полущит/полный щит) над юнитом.
- **Стойка юнита отражает укрытие** (визуал, GDD §12.1): у half cover юнит
  приседает, у full cover — стоит, прижавшись; на открытой местности — обычная
  боевая стойка. Драйвится делегатом `OnCoverStateChanged` в AnimBP.
- На ASC юнита висит тег `Cover.Half` / `Cover.Full` (GameplayEffect).

### 5.6 Overwatch (режим наблюдения)
- Активация: 1 AP, завершает активацию юнита, вешает тег `State.Overwatch`.
- В фазу противника: первый враг, **вошедший в зону видимости** юнита,
  получает реакционный выстрел (штраф −10 к точности, укрытие цели учитывается).
- 1 реакционный выстрел за ход; после выстрела режим снимается.
- Ограничение прототипа: реагирует на вход в зону видимости (стимул
  AIPerception), перемещение целиком внутри зоны выстрел не провоцирует.

### 5.7 Цель миссии, таймер ходов и эвакуация
- **Цель-объект** (`ABombObjective`): бомба у ретранслятора. Юнит вплотную к
  объекту тратит 1 AP на действие «Обезвредить»; нужно **2 таких действия**
  (можно одним юнитом за два хода или двумя юнитами за один). Прогресс виден
  в HUD («Обезвреживание: 1/2»).
- **Таймер ходов** (только миссия): бомба взрывается в конце N-го хода игрока —
  поражение. N по сложности (§10). Счётчик крупно в HUD.
- **Зона эвакуации** (`AEvacZone`, маркер-дым как в XCOM): активируется после
  обезвреживания. Юнит внутри зоны получает действие «Эвакуация» (1 AP) — юнит
  покидает поле (исчезает, помечен «эвакуирован»).
- **Победа миссии** = бомба обезврежена И все живые юниты эвакуированы.
  Тяжелораненые: должны быть подняты медиком и эвакуированы; неэвакуированный
  Downed-юнит на момент последней эвакуации считается потерянным (статистика),
  но миссию не проваливает.
- В туториале зона эвакуации появляется после последнего шага обучения и
  завершает уровень тем же механизмом (без таймера).

### 5.8 Скриптованные ситуации туториала (механика)
Туториал управляет боем поверх общих правил (реализация — level BP + готовые API):
- **Форс-выстрелы**: скрипт вызывает `ResolveShot` с BaseHitChance 100 (первый
  выстрел врага по нам — гарантированное попадание «вы не в укрытии») или 0
  (промах по укрытому юниту). AI врагов в туториале находится под контролем
  скрипта: враги активируются пошагово по сценарию.
- **Заскриптованный раненый**: союзник спавнится сразу в состоянии Downed
  (API `SetDowned(true)`) для урока «оживление».
- Шаги = objectives квест-системы; следующий открывается по игровому триггеру.
  Полный сценарий — [02_LORE_SCRIPT.md](02_LORE_SCRIPT.md) §4.

## 8. Противники и AI

- Один архетип — **«Мародёр»** (BP-наследник `AUnitBase`): винтовка, 20 HP
  урона, BaseAim 65, HP по сложности (см. §10).
- **Модель тревоги — как в XCOM** (green/yellow/red alert; реализовано в
  `AUnitAIController`, enum `EUnitAlertState`):
  - **Patrol (green)** — противник не обнаружен: юнит ходит по своему маршруту
    `PatrolPoints` (расставляются на карте) или стоит на посту. Позиции отряда
    не знает (не читерит).
  - **Investigate (yellow)** — слышал шум боя (каждый выстрел оповещает врагов
    в радиусе 2000) или потерял цель из виду: идёт к последней известной точке;
    дойдя и никого не найдя — успокаивается до Patrol.
  - **Combat (red)** — видит бойца (AIPerception Sight): цель в дальности →
    выстрел (1 AP), далеко → сближение с целью. Приоритет цели: провоцирующий
    танк (тег State.Taunting) > ближайший видимый.
- Все перемещения AI ограничены тем же бюджетом пути (MoveRange за 1 AP),
  что и у игрока; после перемещения пересчитывается укрытие.
- В туториале враги под контролем скрипта (§5.8), параметры занижены
  (BaseAim 40, урон 10) — кроме форс-выстрелов.
- Улучшения (P2, по остатку времени): выбор точки перемещения С УКРЫТИЕМ от
  цели; scamper-рывок в укрытие при первом обнаружении (XCOM pod activation).

## 9. Боевой HUD (WBP_TacticalHUD)

- **Портреты отряда 1–4** (низ слева): HP-бар, пипсы AP (2 шт.), рамка выбранного,
  иконка состояния (overwatch/оборона/ранен/эвакуирован). Клик/цифры 1–4 — выбор.
- **Панель действий** (низ по центру): кнопки из §6 + способность класса,
  серые при нехватке AP/кулдауне; контекстная кнопка «Обезвредить»/«Эвакуация».
- **Инфо о цели** (при наведении на врага): HP и **шанс попадания %**
  (`UTacticsCombatStatics::ComputeHitChance`).
- **Зона хода**: контур достижимости 1 AP / 2 AP у выбранного юнита (§5.3).
- **Индикатор фазы** (верх): «ВАШ ХОД / ХОД ПРОТИВНИКА», номер хода; в миссии —
  **таймер бомбы** («До взрыва: N ходов») и статус цели («Обезвреживание: X/2»).
- **Кнопка «Завершить ход»** (низ справа, хоткей Enter).
- Над юнитами: HP-бары (уже есть, `UUnitHUDWidget`) + иконка укрытия.

## 10. Сложность

Выбирается при новой игре, хранится в `UTacticsSaveGame::Difficulty`, читается
GameMode миссии при спавне врагов.

| Параметр | Лёгкий | Средний | Сложный |
|---|---|---|---|
| Врагов в миссии | 4 | 5 | 6 |
| HP врага | 80 | 100 | 120 |
| BaseAim врага | 55 | 65 | 70 |
| Таймер бомбы (ходов игрока) | 12 | 10 | 8 |

Туториал сложностью не масштабируется (фикс. 4 врага-голограммы, по одному
на секцию A–D сценария).

### 12.1 Анимации

Персонаж и анимации берём из **Game Animation Sample (GASP, Epic)** — он уже
скачан в `D:/UE5/UnrealProjects/GameAnimationSample`. Скелет там UEFN_Mannequin,
но это не проблема (совместимо через ретаргет ЛИБО берём персонажа целиком —
см. ниже два пути). Death/HitReact в GASP нет — их берём из **донора**
(`Content/Characters/Mannequins/Anims/`: Death ×6, HitReact ×8, Fire, Reload).

**Два одобренных пути (выбрать проще по факту в редакторе, этап 4):**
1. **Взять персонажа GASP целиком** (проще): мигрировать `SKM_UEFN_Mannequin`
   + его AnimBlueprint + нужные AnimSequence в наш проект, использовать как меш
   юнита. Донорские Death/HitReact доретаргетить на UEFN-скелет (или заменить
   близкими из GASP). Минус — тянет чуть больше ассетов.
2. **Ретаргет отдельных анимаций** на наш `SK_Mannequin` через готовый
   `RTG_UEFN_to_UE5_Mannequin` (лежит в GASP). Минус — ручной ретаргет пачки.

**Motion matching / PoseSearch из GASP НЕ переносим** — для пошаговой тактики
достаточно стейт-машины (юниты ходят короткими приказами), MM избыточен.

**Правило стоек** (ABP_Soldier, драйвится `OnCoverStateChanged` + тегами):
- Открыт → боевая стойка (rifle idle).
- Half cover → **присед** (crouch idle GASP), перемещение — crouch walk.
- Full cover → стоя, прижат к стене (rifle idle + лёгкий lean — упрощение).
- Downed → лежит (Death-поза с паузой), Overwatch → ADS-поза.
