## ⚠️ ТРЕБУЕТСЯ ОТ ВАС РУКАМИ (агент не может — редактор/PIE)

### A. Пробел завершает ход вместо атаки — проверить назначение IA
Диагностика через MCP: в `IMC_Tactical` пробел = `IA_Attack` (верно), Enter =
`IA_EndTurn`, конфликта клавиш нет, граф BP-контроллера пуст. Значит `IA_Attack`
в BP лежит **не в том поле**. Открыть **BP_TacticalPlayerController → Class
Defaults → категория Input** и проверить:
- [ ] **Attack Action** = `IA_Attack` (НЕ пусто, НЕ другое);
- [ ] **End Turn Action** = `IA_EndTurn` (НЕ `IA_Attack` — если тут по ошибке
      стоит IA_Attack, пробел и завершает ход);
- [ ] скомпилировать BP, проверить: пробел открывает прицел (0 AP), второй
      пробел — выстрел, Enter — конец хода.

### B. Враги не ищут укрытия — это КОНТЕНТ карты, проверить логом
Логика есть (`FindCoverPoint`). Враги «просто стоят/стреляют», потому что на
демо-карте нет укрытий, которые они распознают. Проверить:
- [ ] в консоли PIE (`~`) ввести **`xru1.AI.LogCombat 1`**, сыграть ход врага,
      смотреть Output Log. Строка `[AI] …: открыт=1, укрытие_найдено=0` =
      **на карте нет укрытий рядом** (не баг кода);
- [ ] укрытие для AI = **статик-меш `WorldStatic` высотой ≥60 см** (low=half,
      ≥150=full) в радиусе 120 см от позиции. Расставить ящики/стены между
      врагами и зоной боя — тогда `укрытие_найдено=1` и враг займёт их;
- [ ] если лог показывает `укрытие_найдено=1`, а враг не бежит — это баг, писать.

### C. Баннер прицеливания — положить текст в HUD (опционально)
C++ показывает/прячет виджет `TargetingBanner` по режиму. Чтобы был текст:
- [ ] в **WBP_TacticalHUD** добавить виджет с именем **`TargetingBanner`**
      (Text/Border), текст: «ПРОБЕЛ — огонь · Tab — цель · ПКМ/Esc — отмена».
      Нет виджета — всё работает и без него (BindWidgetOptional).

### D. Тюнинг кадра камеры (опционально, если композиция не нравится)
- [ ] BP камеры (`BP_TacticalCameraPawn`) → категория **`Tactics|Camera|Shot`**:
      `ShotFramePitch` (наклон), `ShotFrameZoom` (дистанция), `ShotFrameTargetBias`
      (0=на стрелке…1=на цели), `ShotFrameLookHeight`, `ShotFrameYawOffset`.

## Проверить в PIE-прогоне (свежие правки, не проверены живьём)

### Флоу атаки
- [ ] пробел: 1-е нажатие — прицел (камера «из-за плеча» показывает **и
      стрелка, и врага**), 2-е нажатие — выстрел; ПКМ/Esc — отмена;
- [ ] **камера возвращается ровно как была** после отмены И после выстрела
      (позиция/поворот/наклон/зум/высота) — нигде не «зависает»;
- [ ] Tab/Q/E листают цели; ЛКМ по выбранной цели — выстрел, по другой —
      перевод прицела; взятая цель подсвечена и не гаснет при уходе курсора;
- [ ] переключение атака↔способность (R во время прицела) не ломает камеру;
- [ ] автопереход бойца/конец хода не рвёт кадр выстрела (ждёт его конца).

### Процентовка (GDD §5.4)
- [ ] шанс зависит от дистанции (в упор > вдали, не 75 везде);
- [ ] стрельба с уступа ≥1.5 м вниз даёт +20;
- [ ] выстрел через щель на стыке мешей больше не проходит (сферы 15 см);
- [ ] сквозь своих стрелять можно; из-за угла укрытия (step-out) — можно.

### Автопереход хода
- [ ] AP кончились у всех → ход сам уходит врагу (не надо жать «Завершить»);
- [ ] боец, потративший последний AP на бег, завершает ход по прибытии.
